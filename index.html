HTML

  • Enter your HTML notes here
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • You can created ordered lists using ol or unordered lists using ul.
  • Add items to these lists using li
  • The meta tag is used for information in the head that doesn't show up on the webpage but effects formatting.
  • You can define different sized headings using h1, h2, h3, etc., with larger numbers equating to a smaller heading.
  • Non list text can be contained in paragraphs using p.

CSS

  • Enter your CSS notes here
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A CSS file must be linked in the html head in order to effect the content.
  • Colors can be semantic or can be the actual hex value.
  • Sections of a CSS file such as ul or li will globally effect those elements unless otherwise specified

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: Adds current branch to the staging area
  • git checkout branch-name: Switches back to the named branch
  • git pull origin main: Creates a pull request to the main branch

Javascript

  • Create a comment in Javascript using // or /* */
  • Define a variable using var. This can have a string, number, boolean, null or undefined value.
  • Strings are what they sound like, strings of letters.
  • Numbers can have values between 2^-1074 and 2^1024, or the negatives of those values, or 0.
  • Booleans can have one of two values, true or false.
  • Control flow is the order in which a computer executes code in a script.
  • For loops can cycle through options
